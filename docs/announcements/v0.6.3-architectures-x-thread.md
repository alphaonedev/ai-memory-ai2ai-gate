```
T1 — Single node, single agent.

One Rust process. SQLite + WAL + FTS5 + HNSW. No network. Zero ops. The bedrock primitive every laptop runs.

97.8% R@5 on LongMemEval (ICLR 2025). Production-ready today.

```

```
Production-ready today.

Diagram → https://alphaonedev.github.io/ai-memory-mcp/architectures-t2.html
```

```
T3 — Multi-node cluster.

Beta — shipped + tested, not for unattended production fleets.

Diagram → https://alphaonedev.github.io/ai-memory-mcp/architectures-t3.html
```

## Follow-up: og:image meta tags for tier pages

Currently the architectures-tN.html pages have `og:title` + `og:description`
but NO `og:image`. X link cards will display text-only previews instead of
showing the architectural diagram thumbnail.

To upgrade link previews, follow-up PR should:

1. Generate static-frame PNG screenshots of each tier's SVG diagram
   (1200 × 630px recommended for X / Open Graph).
2. Commit as `docs/og/architectures-t{1..5}.png` + `architectures.png`.
3. Add to each tier page's `<head>`:

   ```html
   <meta property="og:image" content="https://alphaonedev.github.io/ai-memory-mcp/og/architectures-tN.png">
   <meta property="og:image:width" content="1200">
   <meta property="og:image:height" content="630">
   <meta name="twitter:card" content="summary_large_image">
   <meta name="twitter:image" content="https://alphaonedev.github.io/ai-memory-mcp/og/architectures-tN.png">
   ```

4. Re-deploy. Force X to re-fetch via https://cards-dev.twitter.com/validator
   for each tier URL.

This is an enhancement, not a blocker. Thread is publishable as-is.
